  <name>Task 1: Create environment files and deploy to Vercel</name>
  <files>
    .env.production
    .gitignore
  </files>
  <action>
1. Create .env.production with placeholder (will be set in Vercel dashboard):
   ```
   # Production environment variables
   # These are set in Vercel Dashboard -> Settings -> Environment Variables

   NEXT_PUBLIC_SITE_URL=https://romaix.ai
   # RESEND_API_KEY is set in Vercel dashboard (secret)
   ```

2. Ensure .gitignore includes:
   - .env.local
   - .env.production.local
   (but NOT .env.production - this file documents required variables)

3. Run Vercel CLI deployment:
   ```bash
   npx vercel
   ```
   - If not logged in, CLI will prompt for login
   - Select "Y" to link to existing project or create new
   - Accept defaults for project settings
   - Deployment will start automatically

4. After initial deployment, add environment variables via Vercel CLI:
   ```bash
   npx vercel env add RESEND_API_KEY production
   # (paste API key from existing .env.local when prompted)

   npx vercel env add NEXT_PUBLIC_SITE_URL production
   # (enter: https://romaix.ai)
   ```

5. Trigger production deployment:
   ```bash
   npx vercel --prod
   ```

NOTE: If user is not logged into Vercel CLI, an auth gate will be created. The CLI will provide a login URL.
